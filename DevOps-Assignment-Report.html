üîß Key Challenges & Solutions

Challenge: LocalDateTime Serialization in Redis

Problem: Redis caching failed with LocalDateTime serialization errors, causing 500 Internal Server errors when adding favorites.

Root Cause: Jackson couldn't serialize LocalDateTime fields without JSR310 module.

Solution Implemented:

// Added Jackson JSR310 dependency <dependency> <groupId>com.fasterxml.jackson.datatype</groupId> <artifactId>jackson-datatype-jsr310</artifactId> </dependency> // Enhanced RedisConfig with custom ObjectMapper ObjectMapper objectMapper = new ObjectMapper(); objectMapper.registerModule(new JavaTimeModule()); objectMapper.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);

Result: ‚úÖ Redis caching now works perfectly with LocalDateTime fields

Challenge: Multi-VM Network Configuration

Problem: Backend VM couldn't connect to Redis VM (100% packet loss).

Solution: Configured proper GCP internal networking and Redis bind settings.

Result: ‚úÖ Seamless communication between VMs

Challenge: CI/CD Pipeline Optimization

Problem: Deployment was overwriting Redis configuration.

Solution: Modified deploy.yml to preserve application.properties.

Result: ‚úÖ Deployments maintain Redis settings
